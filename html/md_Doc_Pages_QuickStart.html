Quick Start

KIG is freely available under MIT License terms.

Two implementations of KIG are available: shared library and Docker container. If you work on a personal machine (laptop, PC, mini-server) and you have privileged access (i.e. sudoer) to that resource, you are quite free of choosing the solution that suits your use-case best. For shared clusters, the best solution is Docker, which offers to users the minimal installation of the software and gives them the ability of tailoring the environment as they need.

As a rule of thumb, if you don't feel comfortable building the software from source or if you don't have privileged access to the resource you're working on, go for the containerized solution.

System pre-requisites for Build

To build the library, your machine should have:

  • g++ supporting c++17 standard
  • cmake (min.required version 3.20)

To check if your machine satisfies the requirements, you can launch

g++ --version
cmake --version

in a terminal window.

In case you're not meeting the requirements, consult your distro manual to find out to gracefully set your environment.

Library pre-requisites

The library relies on only one external library which is provided here: https://github.com/ToruNiina/toml11

we suggest cloning the repo and build the software with cmake.

git clone <REPO>
cd <CLONED_REPO_FOLDER>
mkdir build
cd build
cmake ..
make install

Download KIG

In order to download KIG, please enter the following commands on your terminal.

mkdir <dir_name>
cd <dir_name>
git init
git clone "<LINK>"

Test download and Install KIG

Given the previous step was successfully completed, before installing the library it is advisable to test the build.

cd <KIG>
mkdir build
cd build
cmake ..
make all

It should bring 0 warnings and 0 errors.

Install KIG library

Once the build has been tested, installing KIG only requires these two commands:

sudo make install
sudo ldconfig

Run KIG from a container

First, make sure Docker is properly installed on your machine. If not, refer to Docker official website in order to find out how to install it on your distro.

Once Docker is installed, you can pull the container with:

some code

Create a personal Docker volume. You will need it in order to upload scripts and configuration files easily.

docker-volume here

Once you're done with these steps, run the container attaching the docker volume:

run docker run

You can now run your KIG monitoring application without further installs.
