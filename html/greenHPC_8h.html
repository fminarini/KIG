std::vector< double > &  mem_data,
HWconfig &  hw,
double  et 
)

This function uses CPU usage, RAM usage and infrastructure parameters to provide an estimation of the Carbon footprint of a running process on a Linux environment.

This function extracts the carbon footprint of the process as defined in [1] The value returned is measured in gCO2e.

Parameters
[in]cpu_dataA vector containing the CPU usage factor over the execution time.
[in]mem_dataA vector containing the size of RAM allocated by process over execution time
[in]hwAn HWconfig object.
[in]etThe elapsed computing time
Returns
X: The value in gCO2e of the carbon footprint of process.

Before execution:

  • cpu_data is TRUE and cpu_data.size() == 0
  • mem_data is TRUE and mem_data.size() == 0
  • hw is TRUE
  • et is TRUE and != 0

After execution:

  • The carbon footprint X is returned, it should be a non-zero value.

double CPUusage ( CPUsage &  c,
HWconfig &  hw 
)

This function converts CPUsage information from jiffies (clock ticks) to seconds and evaluates the elapsed computing time.

Starting from the data saved in a CPUsage object, this function converts jiffies (clock ticks time-unit) to seconds using the internal CLK_TCK parameter. The elapsed computing time is then saved into the CPUsage object, and the CPU usage factor is returned.

Parameters
[in]cA CPUsage object.
[in]hwAn HWconfig object.
Returns
cpu_usage: A vector containing the sampled cpu usage over computing time

Before execution:

  • c should be TRUE.
  • hw should be TRUE. After execution:

c still TRUE.

  • hw still TRUE.
  • cpu_usage should be bound (0,1]. 0 should correspond to a purely sleeping process and therefore is considered not included in the range.

◆ fetchMem()

double fetchMem ( std::string  PATH_MEM)

This function fetches how much RAM is allocated by the process and returns the value in GB.

Parameters
[in]PATH_MEMThe path at which the file containing info about allocated ram is stored. Typically: "/proc/<pid>/status"
Returns
vmsize: The size (in GB) of allocated RAM

the allocated RAM is considered to be the significant parameter for memory energy consumption in [1]. At the time of execution of the function:

  • PATH_MEM should be TRUE.

At the end of execution of the function:

  • if the monitored process has finished, PATH_MEM will turn FALSE as the PID will get retired.

otherwise, PATH_MEM will still be TRUE

[1] Lannelongue, L. et al. "Green Algorithms: Quantifying the Carbon Footprint of Computation", Advanced Science, 2021.*

This function generates the LaTeX report table for carbon footprint of the experiment. The format of the table is: Experiment Name | Hardware Specs | Power | Elapsed time | CO2e(g) | Cost

Parameters
[in]hwA HWconfig object containing infrastructure data
[in]etElapsed time of the computation
[in]footprintThe carbon footprint of the computation evaluated in carbonFootprint

The function is OPTIONAL. After running the function

std::filesystem::exists("report.txt")

will in any case be TRUE.

In the file, the user will find:

  • LaTeX package dependencies to be included in LaTeX report (if not already included by the user).
  • LaTeX code following the specified layout.
  • One row for each run of our experiment. Data will be reported with 2 decimal precision, where needed.
